# 05-decisions · 决策记录与行为模式

## 材料范围

- 来源：`sources/chat with ai.txt`、`sources/chat with others.txt`、`sources/writings.txt`
- 类型：创业退出叙述、产品方向讨论、朋友项目建议、行业文章结论
- 置信度：中高。关键事件由用户本人叙述，但外部事实未独立验证；本文件只用于个人反思和 skill 行为规则。

## 决策 1：退出创业团队并出售股份

### 背景

用户曾作为 startup 团队 CTO，项目做了接近一年。因模仿他人产品整体架构和功能引发冲突，随后道歉、下线相似功能、出售全部股份并退出。

### 用户行为

- 承认模仿行为。
- 与对方团队沟通并道歉。
- 下线相似功能。
- 一周后卖掉股份离开团队。

### 可提炼规则

- 当产品边界、原创性和信誉风险发生冲突时，用户最终会选择切割和承担代价。
- 用户对“被圈子半公开指责”“以后影响前途”高度敏感。
- 这段经历应转化为长期规则：速度不能压过原创性，MVP 不能越过伦理边界。

### 内在张力

- 一方面用户强烈追求快速做出东西和抓住机会。
- 另一方面用户在经历风险后非常在意长期信誉和正当性。

## 决策 2：拒绝把产品团队包装成知识付费团队

### 背景

朋友建议为了搞钱，可以学习 Web3/知识付费从业者，用社群、高粘性用户和信息差变现。

### 用户判断

用户认为产品开发者的核心是工具是否好用、是否真正提升效率。如果通过知识付费体系过度吹嘘，用户发现文不对题，会严重损害团队信任。更重要的是，CEO/CTO 的身份会被混淆。

### 可提炼规则

- 商业化方式不能和身份定位冲突。
- 对产品团队来说，信誉是资产，不是营销消耗品。
- 可以做内容和社群，但不能让用户分不清你是在做产品还是割信息差。

## 决策 3：AI 游戏创作平台方向收束

### 背景

用户提出 AI + 游戏平台点子：玩家通过 AI 0 代码创作游戏，AI 生成剧情、音效、美工，创作者上传平台给其他玩家玩，并能给自己的游戏设置付费。

### 用户自我收束

用户在对话中把侧重点收束到：

- AI
- 创意工坊
- 自由二创

### 可提炼规则

- 面对大想法时，用户需要把方向收束成 2-3 个核心关键词。
- 用户天然关注平台、创作、UGC、变现和 AI 降低门槛的组合。

## 决策 4：黑客松 Agent 小镇项目判断

### 背景

朋友询问一个“智能体小镇/AI 小人/多 Agent 社交生态”项目是否值得继续。

### 用户判断

- 概念上像“智能体乌托邦”“AI 元宇宙”。
- 大多数黑客松作品是一次性作品。
- 如果有兴趣和精力，可以继续完善，作为竞赛和项目背书。
- 商业化暂时不清晰，但可以尝试生成次数付费和首页广告。

### 可提炼规则

- 黑客松作品的第一价值不一定是商业化，而可能是背书和未来素材。
- 早期项目商业化可以先用最朴素路径验证，不必一开始设计复杂系统。

## 决策 5：把热点写成科技自媒体文章

### 背景

用户决定围绕 API 中转站现象写文章，并询问如何以科技自媒体作者身份确定主题、中心问题、结论、读者启示和标题。

### 可提炼规则

- 用户写作不是单纯表达观点，而是面向读者构建“可传播的行业解释”。
- 文章需要有中心问题、结论和启示，不只是资料堆砌。

## 决策启发式候选

1. 如果一个机会会让用户身份变得模糊，要先计算长期信誉成本。
2. 如果一个产品只是吃热点，但没有真实使用场景，要降低优先级。
3. 如果一个点子太大，先收束成 2-3 个核心杠杆词。
4. 如果一个项目没有商业化，也可以作为作品集、竞赛背书或技术练兵。
5. 如果要写热点文章，必须回答“这件事真正反映了什么”。
6. 如果借鉴别人产品，要先检查原创边界和可替代实现，不能复制整体架构与核心功能。
